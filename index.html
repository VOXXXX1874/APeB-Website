Introduction

LLM-powered agents struggle with personalization, which requires resolving vanilla intents, inferring user-specific preferences, and finding useroptimal solutions from competing alternatives. Existing benchmarks largely fail to assess these challenges under vague goals and noisy interaction histories. We propose personalized product search (PPS) as a testbed for agentic personalization under raw queries and diverse histories. We construct Agent Personalized Benchmark (APeB) from holistic action logs, featuring rich histories, vanilla intents, and hard candidate sets. We benchmark eight state-of-the-art LLMs with agentic workflows. LLMs excel on user-refined explicit queries, but struggle on initial vague queries that require latent intent and preference discovery. Fine-grained analysis attribute this gap to ineffective use of historical interactions for intent and preference inference. We further find that a simple history-conditioned refinement of vague queries yields consistent gains, emphasizing the role of effective history utilization in personalization.

Data Examples

All data are newly collected from TikTok user data, not from any existing dataset.

Construction Pipeline of APeB: from behavioral logs to benchmark components: curated heterogeneous interactions, non-trivial shopping session discovery (search refinement, extensive browsing, and intent-order alignment), and intent/refined query candidate pair identification.

A concrete example of a behavioral log in APeB, showcasing how can we deduces user intent and preferences from noisy histories to reproduce the decision-making process and make personalized recommendations. While benchmarking different models, we will use history, intent/refined query, and candidate as input and evaluate whether the model choose the correct candidate.

Data Statistics

APeB contains 5K+ shopping sessions that covers various product categories. Each session is equipped with 10+ non-trivial candidates and rich interaction history.

Case Studies

Recommendation from Different Models

Impact of capturing the key preferences from history on recommendation performance. Models need to deduce user preference from history based on the intent query.

An example of of non-triveal shopping session in APeB, where the user refines search queries multiple times and browses extensively before making a purchase. We will see show different models analyze the history and make recommendations based on the inferred intent and preferences.

GPT5.2 successfully capture the user's perference "long battery life" from the history and recommend the correct product.

Gemini 2.5 consider "health monitoring" as the main user intent, but in fact the "health monitoring" feature is not a majority preference in history.

Qwen3 deduce the user preference on "health monitoring" based on the fact that user has browsed body serums and bleaching enhancers, which is somewhat far-fetched and leads to wrong recommendation.

Recommendation from Different Agents

Comparison of different agent workflows on analyzing user history and make personalized recommendation based on intent query.

An example of of non-triveal shopping session in APeB, where the user refines search queries multiple times and browses extensively before making a purchase. We will show how different agent architectures analyze the history.

VQRA agent successfully detect the true intent of "hair growth oil" based on the history and rewrite the intent query to a clearer one, which leads to correct recommendation.

ReAct agent is misled by the ambiguity in user's intent query and the noisy history, which leads to wrong recommendation.
